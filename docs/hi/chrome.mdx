कंटेनर में एक पूर्व-कॉन्फ़िगर [Chrome DevTools MCP](https://github.com/anthropics/chrome-devtools-mcp) सर्वर शामिल है जो Claude Code को हेडलेस ब्राउज़र ऑटोमेशन क्षमताएं प्रदान करता है। वेब पेज नेविगेट करें, स्क्रीनशॉट लें, DOM का निरीक्षण करें, और पेज तत्वों के साथ इंटरैक्ट करें — किसी अतिरिक्त सेटअप की आवश्यकता नहीं।

## यह कैसे काम करता है

एंट्रीपॉइंट पोर्ट 9222 पर रिमोट डीबगिंग के साथ एक स्थायी हेडलेस Chrome लॉन्च करता है। chrome-devtools-mcp MCP सर्वर Claude Code सेटिंग्स में पंजीकृत है और `--browserUrl` के माध्यम से इस साझा ब्राउज़र से कनेक्ट होता है। सभी सेशन एक ही ब्राउज़र इंस्टेंस और कुकी जार साझा करते हैं — लॉगिन, कुकीज़ और प्रमाणीकरण सभी प्रोजेक्ट्स में बने रहते हैं।

## उपलब्ध टूल्स

| टूल | विवरण |
|------|-------------|
| `navigate_page` | किसी URL पर नेविगेट करें और पेज लोड होने की प्रतीक्षा करें |
| `take_screenshot` | वर्तमान पेज का PNG स्क्रीनशॉट कैप्चर करें |
| `take_snapshot` | पेज का एक्सेसिबिलिटी ट्री स्नैपशॉट प्राप्त करें |
| `click` | पेज पर किसी तत्व पर क्लिक करें |
| `type` | इनपुट फ़ील्ड में टेक्स्ट टाइप करें |
| `hover` | किसी तत्व पर होवर करें |
| `scroll` | पेज या किसी विशिष्ट तत्व को स्क्रॉल करें |
| `evaluate` | पेज संदर्भ में JavaScript निष्पादित करें |
| `get_console_logs` | कंसोल लॉग संदेश प्राप्त करें |
| `get_page_info` | पेज मेटाडेटा प्राप्त करें (शीर्षक, URL, आदि) |

## आर्किटेक्चर

ब्राउज़र स्टैक चार घटकों का उपयोग करता है जो एक साथ काम करते हैं:

1. **Playwright Chromium** — ARM64-संगत Chromium बाइनरी जो बिल्ड समय पर `npx playwright install --with-deps chromium` के माध्यम से इंस्टॉल की जाती है
2. **Chrome सिमलिंक** — Puppeteer के अपेक्षित पथ (`/opt/google/chrome/chrome`) को Playwright बाइनरी से जोड़ता है
3. **साझा Chrome प्रक्रिया** — `chrome-browser.sh` द्वारा `--remote-debugging-port=9222` के साथ लॉन्च की जाती है, क्रैश होने पर स्वतः पुनरारंभ होती है
4. **वैश्विक MCP पंजीकरण** — `settings.json` chrome-devtools-mcp को `--browserUrl=http://localhost:9222` के साथ पंजीकृत करता है

Google Chrome के पास कोई ARM64 `.deb` पैकेज नहीं है, और Ubuntu 24.04 का `chromium-browser` apt पैकेज snap पर रीडायरेक्ट करता है (जो कंटेनरों में काम नहीं करता)। Playwright का बंडल किया गया Chromium ARM64 कंटेनरों के लिए एकमात्र विश्वसनीय विकल्प है।

## सत्यापन

कंटेनर के अंदर सेटअप काम कर रहा है इसकी पुष्टि करें:

```bash
# Chrome बाइनरी प्रतिक्रिया देता है
/opt/google/chrome/chrome --version

# सिमलिंक सही है
ls -la /opt/google/chrome/chrome

# रिमोट डीबगिंग सक्रिय है
curl http://localhost:9222/json/version

# सेल्फ-टेस्ट में Chrome जांच शामिल है
claude-self-test
```

## हेडेड मोड (VNC)

जब `ENABLE_VNC=true` होता है, तो एंट्रीपॉइंट Chrome लॉन्च करने से पहले VNC स्टैक शुरू करता है। `chrome-browser.sh` स्क्रिप्ट पता लगाती है कि `DISPLAY` सेट है और Chrome को वर्चुअल डिस्प्ले पर हेडेड मोड में चलाती है, ताकि आप ब्राउज़र को VNC व्यूअर में देख सकें।

वर्चुअल डिस्प्ले देखने के लिए `http://localhost:6080/vnc.html` खोलें। विवरण के लिए [रिमोट डिस्प्ले (noVNC)](./vnc) देखें।

## समस्या निवारण

रिमोट डीबगिंग विफलताओं, प्रोफ़ाइल लॉक त्रुटियों, और डुप्लिकेट MCP टूल्स सहित सामान्य समस्याओं के समाधान के लिए [समस्या निवारण — Chrome DevTools MCP](./troubleshooting#chrome-devtools-mcp) देखें।
