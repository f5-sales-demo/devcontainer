# Architecture

## Container Layout

The environment consists of two Docker containers managed by `docker-compose.yml`:

```
┌─────────────────────────────────────────────────┐
│  Docker Network (devcontainer_default)           │
│                                                  │
│  ┌──────────────────────────┐                    │
│  │  dev                     │                    │
│  │  Ubuntu 24.04            │                    │
│  │                          │ http://proxy:8082  │
│  │  AI tools ──────────────────►┌─────────────┐ │
│  │  Your code               │   │  proxy      │ │
│  │  /workspace              │   │  Python API │ │
│  │  /home/<user>            │   │  translator │ │
│  └──────────────────────────┘   └──────┬──────┘ │
│                                        │        │
└────────────────────────────────────────┼────────┘
                                         │ HTTPS
                                         ▼
                                  Your API Provider
```

### Dev Container

The main container where you work. Built from `Dockerfile` on top of Microsoft's `devcontainers/base:ubuntu-24.04` image.

### Proxy Container

A lightweight Python service built from `Dockerfile.proxy`. It translates API requests between formats, allowing tools that expect one API provider to work with another.

The proxy starts first (health-checked) and the dev container waits for it to be ready.

## Build Layers

Tools are installed in three layers, each with a specific purpose:

### Layer 1: Dockerfile (Image Build Time)

Runs once when you build the image. Contains:

- **System packages** — `apt-get install` for things like ffmpeg, nmap, neovim, poppler-utils
- **Standalone binaries** — Direct downloads for actionlint, yt-dlp
- **User setup** — UID/GID mapping to match your host user

These are baked into the Docker image and available immediately.

### Layer 2: Devcontainer Features (Container Creation)

Applied by VS Code or the `devcontainer` CLI when creating the container. These are [OCI-distributed packages](https://containers.dev/features) that install and configure tools:

| Source | Features |
|--------|----------|
| **Official** (`devcontainers/features`) | common-utils, git, github-cli, node, python, go, rust, java, terraform, kubectl-helm, docker-outside-of-docker, aws-cli, azure-cli, powershell |
| **Community** | claude-code, opencode, codex, uv, prettier, act, terraform-docs, devcontainers-cli |

Features run as root during container creation and handle all the complexity of installing runtimes, setting up PATH, and configuring shell integration.

### Layer 3: Post-Create Script (After Features)

Runs after features are installed, so it has access to the runtimes they provide (Python, Node.js, etc.):

- **pip tools** — pre-commit, ansible, black, pylint, yamllint
- **npm tools** — markdownlint-cli2, openclaw

## Volume Architecture

No host directories are mounted into the container. All data lives in Docker named volumes:

```
Docker Volumes
├── workspace    →  /workspace         (your code)
└── home         →  /home/<username>   (dotfiles, caches, tools)
```

This is a deliberate security choice. See [Security](./security) for details.

### Volume Lifecycle

| Action | workspace | home |
|--------|-----------|------|
| `docker compose down` | ✅ Preserved | ✅ Preserved |
| `docker compose up -d --build` | ✅ Preserved | ✅ Preserved |
| `docker compose down -v` | ❌ Deleted | ❌ Deleted |

## Startup Sequence

```
1. docker compose up -d
   │
   ├─ proxy container starts
   │  └─ Health check: HTTP GET http://localhost:8082/
   │
   └─ dev container starts (after proxy is healthy)
      └─ entrypoint.sh runs:
         ├─ Fix volume permissions
         ├─ Configure git (name, email)
         ├─ Install SSH key (if provided)
         └─ Seed Claude Code config
            │
            └─ exec sleep infinity (keeps container running)

2. VS Code / devcontainer CLI connects
   │
   ├─ Devcontainer features install (Layer 2)
   │
   ├─ postCreateCommand runs (Layer 3)
   │  ├─ pip install: pre-commit, ansible, black, pylint, yamllint
   │  └─ npm install: markdownlint-cli2, openclaw
   │
   └─ postStartCommand runs
      └─ Verifies proxy connectivity and lists installed tools
```

## Network

The two containers share a Docker bridge network (`devcontainer_default`). The dev container reaches the proxy at `http://proxy:8082` using Docker's built-in DNS.

The proxy exposes port `18082` on the host's loopback interface (`127.0.0.1`) for debugging. This is not required for normal operation.

## User Identity

The container user is created to match your host user:

| Build Arg | Purpose | Default |
|-----------|---------|---------|
| `USERNAME` | Container username | `vscode` |
| `USER_UID` | User ID | `1000` |
| `USER_GID` | Group ID | `1000` |

This ensures files created inside the container have the correct ownership if you ever need to inspect the Docker volumes from the host.
