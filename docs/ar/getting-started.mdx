## 2. أضف بيانات المصادقة (اختياري)

أنشئ ملف `.env` في نفس المجلد لتكوين Claude Code مسبقًا. اختر أحد الخيارات التالية:

### الخيار أ: Claude Max (OAuth)

```ini
CLAUDE_CODE_OAUTH_TOKEN=sk-ant-oat01-your-token-here
```

### الخيار ب: LiteLLM أو وكيل متوافق مع Anthropic

إذا كان لديك نسخة LiteLLM أو وكيل آخر متوافق مع Anthropic، قم بتعيين هذين المتغيرين:

```ini
LITELLM_BASE_URL=https://your-litellm-endpoint.example.com
LITELLM_API_KEY=your-api-key
```

قم بتعيين النطاق فقط — تشتق الحاوية لاحقات URL الخاصة بالمزود تلقائيًا (مثل `/anthropic` لـ Claude Code).

### الملء التلقائي من بيانات الاعتماد المحلية

إذا كان لديك `git` و`gh` مكوّنان على جهازك المضيف، يمكنك ملء معظم قيم `.env` تلقائيًا. ابدأ بنسخ القالب:

```bash
cp .env.example .env
```

ثم نفّذ الأوامر التي تتوافق مع إعدادك:

```bash
# Git identity
echo "GIT_AUTHOR_EMAIL=$(git config user.email)" >> .env
echo "GIT_AUTHOR_NAME=\"$(git config user.name)\"" >> .env

# Timezone
echo "TZ=$(readlink /etc/localtime | sed 's|.*/zoneinfo/||')" >> .env

# GitHub CLI token (requires: gh auth login)
echo "GH_TOKEN=$(gh auth token)" >> .env

# SSH key (if you use SSH for git)
echo "SSH_PRIVATE_KEY=$(base64 < ~/.ssh/id_ed25519)" >> .env
```

ثم حرّر `.env` لإضافة رمز OAuth أو إعدادات وكيل LiteLLM (انظر الخيارات أعلاه).

:::tip[مساعدو الذكاء الاصطناعي]
يحتوي ملف `.env.example` على علامات `@auto-detect` على كل متغير يمكن ملؤه من الأدوات المحلية. يمكن لمساعدي الذكاء الاصطناعي مثل Claude Code مسح هذه العلامات وتنفيذ الأوامر المرتبطة لبناء ملف `.env` كامل تلقائيًا.
:::

## 7. العرض عن بُعد (noVNC)

تتضمن الحاوية شاشة افتراضية لمشاهدة وكلاء الذكاء الاصطناعي وهم يتحكمون في المتصفح. افتح `http://localhost:6080/vnc.html` للاتصال. انظر [العرض عن بُعد (noVNC)](./vnc) للتفاصيل.

## 8. أتمتة المتصفح (Chrome DevTools MCP)

يمكن لـ Claude Code التنقل في صفحات الويب والتقاط لقطات شاشة وفحص DOM باستخدام خادم Chrome DevTools MCP المدمج. لا حاجة لأي إعداد — يعمل تلقائيًا في الوضع بدون واجهة رسومية. انظر [Chrome DevTools MCP](./chrome) للتفاصيل.

## 9. الإيقاف وإعادة التشغيل

<Tabs syncKey="runtime">
<TabItem label="Docker">

```bash
# إيقاف (يحافظ على البيانات)
docker compose down

# البدء مجددًا
docker compose up -d

# حذف كل شيء والبدء من جديد
docker compose down -v
docker compose up -d
```

</TabItem>
<TabItem label="Podman">

```bash
# إيقاف (يحافظ على البيانات)
podman-compose down

# البدء مجددًا (السحب يضمن أحدث صورة)
podman-compose pull && podman-compose up -d

# حذف كل شيء والبدء من جديد
podman-compose down -v
podman-compose pull && podman-compose up -d
```

</TabItem>
</Tabs>

يتم الاحتفاظ بالكود الخاص بك في `/workspace` والمجلد الرئيسي عبر عمليات إعادة التشغيل. انظر [التكوين — استمرارية البيانات](./configuration#data-persistence) للتفاصيل.
