## التثبيت بأمر واحد

```bash
curl -fsSL https://f5xc-salesdemos.github.io/devcontainer/scripts/install.sh | bash
```

يتولى هذا الأمر الواحد عملية الإعداد الكاملة لأول تشغيل:

1. تثبيت حزم Homebrew (podman, podman-compose, gh, iterm2, nerd fonts)
2. تكوين iTerm2 (الخط، التنبيه الصوتي، سجل التمرير، السمة الداكنة، اختصار Shift+Enter)
3. تهيئة وضبط آلة Podman (ذاكرة 16 جيجابايت، 4 وحدات معالجة، قرص 220 جيجابايت، zram، swappiness)
4. تنزيل `docker-compose.yml` و `devcontainer.sh` وقالب `.env` إلى الدليل الحالي

:::note[مطلوب Homebrew]
إذا لم يكن Homebrew مثبتاً، يتوقف السكربت مع عرض التعليمات.
قد يوفر قسم تكنولوجيا المعلومات لديكم Homebrew كبرنامج مُدار — تحقق معهم أولاً.
:::

## بعد التثبيت

عدّل `.env` بإدخال بيانات اعتمادك:

```bash
vi .env
```

ثم ابدأ تشغيل الحاوية:

```bash
./devcontainer.sh
```

يكتشف السكربت منطقتك الزمنية تلقائياً ويسحب رمز GitHub جديداً من `gh auth token` عند كل تشغيل.

## ما يتم إنشاؤه

بعد تشغيل المثبّت، سيحتوي دليلك على:

| الملف | الغرض |
|---|---|
| `devcontainer.sh` | سكربت التشغيل (شغّل هذا لبدء الحاوية) |
| `docker-compose.yml` | تعريف خدمة الحاوية |
| `.env` | بيانات اعتمادك (من القالب، عدّله قبل أول تشغيل) |

## عمليات التشغيل اللاحقة

```bash
./devcontainer.sh
```

## الاتصال

```bash
podman exec -it devcontainer zsh
```

## التحديث

اسحب أحدث صورة وأعد التشغيل:

```bash
podman compose pull
./devcontainer.sh
```

## البناء المحلي

ابنِ الصورة محلياً من المصدر (arm64 فقط):

```bash
./devcontainer.sh build
```
