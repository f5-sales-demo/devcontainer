يتضمن الحاوية خادم [Chrome DevTools MCP](https://github.com/anthropics/chrome-devtools-mcp) مُعدّ مسبقًا يمنح Claude Code إمكانيات أتمتة المتصفح بدون واجهة رسومية. يمكنك التنقل بين صفحات الويب والتقاط لقطات الشاشة وفحص DOM والتفاعل مع عناصر الصفحة — دون الحاجة إلى أي إعداد إضافي.

## كيف يعمل

يُطلق نقطة الدخول متصفح Chrome مستمر بدون واجهة رسومية مع تصحيح أخطاء عن بُعد على المنفذ 9222. يتم تسجيل خادم chrome-devtools-mcp MCP في إعدادات Claude Code ويتصل بهذا المتصفح المشترك عبر `--browserUrl`. تتشارك جميع الجلسات نسخة متصفح واحدة وملف تعريف ارتباط واحد — تبقى عمليات تسجيل الدخول وملفات تعريف الارتباط والمصادقة مستمرة عبر المشاريع.

## الأدوات المتاحة

| الأداة | الوصف |
|------|-------------|
| `navigate_page` | الانتقال إلى عنوان URL والانتظار حتى يتم تحميل الصفحة |
| `take_screenshot` | التقاط لقطة شاشة بصيغة PNG للصفحة الحالية |
| `take_snapshot` | الحصول على لقطة شجرة إمكانية الوصول للصفحة |
| `click` | النقر على عنصر في الصفحة |
| `type` | كتابة نص في حقل إدخال |
| `hover` | التمرير فوق عنصر |
| `scroll` | تمرير الصفحة أو عنصر محدد |
| `evaluate` | تنفيذ JavaScript في سياق الصفحة |
| `get_console_logs` | استرجاع رسائل سجل وحدة التحكم |
| `get_page_info` | الحصول على البيانات الوصفية للصفحة (العنوان، عنوان URL، إلخ) |

## البنية المعمارية

تستخدم حزمة المتصفح أربعة مكونات تعمل معًا:

1. **Playwright Chromium** — ملف Chromium الثنائي المتوافق مع ARM64 يتم تثبيته أثناء البناء عبر `npx playwright install --with-deps chromium`
2. **رابط رمزي لـ Chrome** — يربط المسار المتوقع من Puppeteer (`/opt/google/chrome/chrome`) بملف Playwright الثنائي
3. **عملية Chrome مشتركة** — يتم تشغيلها بواسطة `chrome-browser.sh` مع `--remote-debugging-port=9222`، وتعيد التشغيل تلقائيًا عند التعطل
4. **تسجيل MCP عام** — يسجل `settings.json` خادم chrome-devtools-mcp مع `--browserUrl=http://localhost:9222`

لا تمتلك Google Chrome حزمة `.deb` لمعمارية ARM64، وحزمة `chromium-browser` في Ubuntu 24.04 عبر apt تُعيد التوجيه إلى snap (الذي لا يعمل في الحاويات). يُعد Chromium المُجمّع مع Playwright الخيار الوحيد الموثوق لحاويات ARM64.

## التحقق

تأكد من أن الإعداد يعمل داخل الحاوية:

```bash
# Chrome binary responds
/opt/google/chrome/chrome --version

# Symlink is correct
ls -la /opt/google/chrome/chrome

# Remote debugging is active
curl http://localhost:9222/json/version

# Self-test includes Chrome checks
claude-self-test
```

## الوضع مع الواجهة الرسومية (VNC)

عندما يكون `ENABLE_VNC=true`، تبدأ نقطة الدخول حزمة VNC قبل تشغيل Chrome. يكتشف سكريبت `chrome-browser.sh` أن `DISPLAY` مُعيّن ويشغّل Chrome في الوضع مع الواجهة الرسومية على الشاشة الافتراضية، حتى تتمكن من رؤية المتصفح في عارض VNC.

افتح `http://localhost:6080/vnc.html` لرؤية الشاشة الافتراضية. راجع [العرض عن بُعد (noVNC)](./vnc) للتفاصيل.
