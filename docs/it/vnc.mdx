Il container esegue uno stack di display virtuale (Xvfb + x11vnc + noVNC + fluxbox) che permette di osservare gli agenti AI mentre controllano un browser e di intervenire quando necessario — ad esempio, per gestire le richieste di login o MFA.

## Connessione

1. Apri `http://localhost:6080/vnc.html` nel tuo browser e clicca su **Connect**
2. Vedrai un desktop fluxbox leggero — questo è il display virtuale del container

Gli agenti AI che utilizzano `@playwright/mcp` in modalità con interfaccia grafica rendono Chromium in questo display. Puoi cliccare e digitare direttamente nella finestra noVNC per intervenire in qualsiasi momento — non è necessario mettere in pausa o riprendere.

## Avvio Manuale del Browser

Per aprire un browser manualmente all'interno del container:

```bash
chromium --no-sandbox &
```

Per eseguire i test di Playwright in modalità con interfaccia grafica:

```bash
npx playwright test --headed
```

## Variabili d'Ambiente

| Variabile | Default | Descrizione |
|----------|---------|-------------|
| `ENABLE_VNC` | `true` | Imposta su `false` per disabilitare lo stack VNC |
| `VNC_RESOLUTION` | `1280x1024x24` | Risoluzione del display virtuale (LxAxProfondità) |
| `DISPLAY` | `:99` | Numero del display X11 |
| `NOVNC_HOST_PORT` | `6080` | Porta lato host per noVNC (modifica per evitare conflitti con altri container) |

Imposta queste variabili nel tuo file `.env` o in `.devcontainer/devcontainer.json` sotto `containerEnv`.

## Disabilitare lo Stack VNC

Per operazioni solo in modalità headless, imposta `ENABLE_VNC=false` nel file `.env` e riavvia il container:
