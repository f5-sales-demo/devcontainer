## Prerequisiti

### Runtime per Container

Hai bisogno di un runtime per container sulla tua macchina. Docker o Podman funzionano entrambi.

<Tabs syncKey="runtime">
<TabItem label="Docker">

| Piattaforma | Installazione |
|-------------|---------------|
| macOS | [Docker Desktop per Mac](https://docs.docker.com/desktop/install/mac-install/) |
| Windows | [Docker Desktop per Windows](https://docs.docker.com/desktop/install/windows-install/) (backend WSL2) |
| Linux | [Docker Engine](https://docs.docker.com/engine/install/) o [Docker Desktop per Linux](https://docs.docker.com/desktop/install/linux/) |

Verifica che funzioni:

```bash
docker --version
docker compose version
```

</TabItem>
<TabItem label="Podman">

| Piattaforma | Installazione |
|-------------|---------------|
| macOS | [Podman Desktop per Mac](https://podman-desktop.io/docs/installation/macos-install) |
| Windows | [Podman Desktop per Windows](https://podman-desktop.io/docs/installation/windows-install) |
| Linux | [Podman](https://podman.io/docs/installation#installing-on-linux) + [Podman Compose](https://github.com/containers/podman-compose#installation) |

Verifica che funzioni:

```bash
podman --version
podman-compose version
```

:::note[Solo Linux]
Podman su Linux esegue i container direttamente (senza VM). Assicurati che il socket Podman sia attivo affinché `podman-compose` possa connettersi:

```bash
systemctl --user enable --now podman.socket
```
:::

</TabItem>

### Risorse di Sistema

| Risorsa | Minimo | Consigliato |
|---------|--------|-------------|
| Disco | 5 GB | 10 GB+ |
| RAM | 8 GB allocati al runtime del container | 12 GB+ |
| CPU | 2 core | 4+ core |

<Tabs syncKey="runtime">
<TabItem label="Docker">

Modifica i limiti delle risorse in Docker Desktop &rarr; Impostazioni &rarr; Risorse.

</TabItem>
<TabItem label="Podman">

Su **macOS/Windows**, modifica i limiti delle risorse in Podman Desktop &rarr; Impostazioni &rarr; Risorse, oppure tramite CLI:

```bash
podman machine set --cpus 4 --memory 12288
podman machine stop && podman machine start
```

Su **Linux**, i container condividono direttamente le risorse dell'host — non è necessaria alcuna configurazione delle risorse.

</TabItem>

## 1. Configurazione

Crea una cartella e scarica il file compose:

```bash
mkdir devcontainer && cd devcontainer
curl -fsSLO https://raw.githubusercontent.com/f5-sales-demo/devcontainer/main/docker-compose.yml
```

## 2. Aggiungi le Tue Credenziali di Autenticazione (Opzionale)

Crea un file `.env` nella stessa cartella per pre-configurare Claude Code. Scegli una delle seguenti opzioni:

### Opzione A: Claude Max (OAuth)

```ini
CLAUDE_CODE_OAUTH_TOKEN=sk-ant-oat01-your-token-here
```

### Opzione B: LiteLLM o proxy compatibile con Anthropic

Se disponi di un'istanza LiteLLM o di un altro proxy compatibile con Anthropic, imposta queste due variabili:

```ini
LITELLM_BASE_URL=https://your-litellm-endpoint.example.com
LITELLM_API_KEY=your-api-key
```

Imposta solo il dominio — il container deriva automaticamente i suffissi URL specifici del provider (ad es. `/anthropic` per Claude Code).

### Popolare automaticamente dalle credenziali locali

Se hai `git` e `gh` configurati sul tuo host, puoi popolare automaticamente la maggior parte dei valori del `.env`. Inizia copiando il template:

```bash
cp .env.example .env
```

Poi esegui i comandi corrispondenti alla tua configurazione:

```bash
# Git identity
echo "GIT_AUTHOR_EMAIL=$(git config user.email)" >> .env
echo "GIT_AUTHOR_NAME=\"$(git config user.name)\"" >> .env

# Timezone
echo "TZ=$(readlink /etc/localtime | sed 's|.*/zoneinfo/||')" >> .env

# GitHub CLI token (requires: gh auth login)
echo "GH_TOKEN=$(gh auth token)" >> .env

# SSH key (if you use SSH for git)
echo "SSH_PRIVATE_KEY=$(base64 < ~/.ssh/id_ed25519)" >> .env
```

Poi modifica il `.env` per aggiungere il tuo token OAuth o le impostazioni del proxy LiteLLM (vedi le opzioni sopra).

:::tip[Assistenti AI]
Il file `.env.example` contiene marcatori `@auto-detect` su ogni variabile che può essere popolata dagli strumenti locali. Gli assistenti AI come Claude Code possono cercare questi marcatori ed eseguire i comandi associati per costruire automaticamente un `.env` completo.
:::

<Tabs syncKey="runtime">
<TabItem label="Docker">

```bash

```

</TabItem>
<TabItem label="Podman">

```bash

claude
```

## 7. Display Remoto (noVNC)

Il container include un display virtuale per osservare gli agenti AI controllare un browser. Apri `http://localhost:6080/vnc.html` per connetterti. Consulta [Display Remoto (noVNC)](./vnc) per i dettagli.

## 8. Automazione del Browser (Chrome DevTools MCP)

Claude Code può navigare pagine web, catturare screenshot e ispezionare il DOM utilizzando il server Chrome DevTools MCP integrato. Non è necessaria alcuna configurazione — funziona automaticamente in modalità headless. Consulta [Chrome DevTools MCP](./chrome) per i dettagli.

## 9. Arresto e Riavvio

<Tabs syncKey="runtime">
<TabItem label="Docker">

```bash
# Arresto (preserva i dati)
docker compose down

# Riavvio
docker compose up -d

# Eliminare tutto e ricominciare da zero
docker compose down -v
docker compose up -d
```

</TabItem>
<TabItem label="Podman">

```bash
# Arresto (preserva i dati)
podman-compose down

# Riavvio (pull assicura l'immagine più recente)
podman-compose pull && podman-compose up -d

# Eliminare tutto e ricominciare da zero
podman-compose down -v
podman-compose pull && podman-compose up -d
```

</TabItem>
</Tabs>

Il tuo codice in `/workspace` e la directory home persistono tra i riavvii. Consulta [Configurazione — Persistenza dei Dati](./configuration#data-persistence) per i dettagli.
