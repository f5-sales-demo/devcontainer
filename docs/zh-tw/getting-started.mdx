:::tip[macOS 快速安裝]
如果您使用搭載 Apple Silicon 的 macOS，請跳過此頁面並使用[一行安裝指令](/devcontainer/install/)：

curl -fsSL https://f5xc-salesdemos.github.io/devcontainer/scripts/install.sh | bash

## 前置需求

### 容器執行環境

您需要在機器上安裝容器執行環境。Docker 或 Podman 皆可使用。

<Tabs syncKey="runtime">
<TabItem label="Docker">

| 平台 | 安裝方式 |
|----------|---------|
| macOS | [Docker Desktop for Mac](https://docs.docker.com/desktop/install/mac-install/) |
| Windows | [Docker Desktop for Windows](https://docs.docker.com/desktop/install/windows-install/)（WSL2 後端） |
| Linux | [Docker Engine](https://docs.docker.com/engine/install/) 或 [Docker Desktop for Linux](https://docs.docker.com/desktop/install/linux/) |

驗證是否正常運作：

```bash
docker --version
docker compose version
```

</TabItem>
<TabItem label="Podman">

| 平台 | 安裝方式 |
|----------|---------|
| macOS | [Podman Desktop for Mac](https://podman-desktop.io/docs/installation/macos-install) |
| Windows | [Podman Desktop for Windows](https://podman-desktop.io/docs/installation/windows-install) |
| Linux | [Podman](https://podman.io/docs/installation#installing-on-linux) + [Podman Compose](https://github.com/containers/podman-compose#installation) |

驗證是否正常運作：

```bash
podman --version
podman-compose version
```

:::note[僅限 Linux]
Podman 在 Linux 上直接執行容器（無虛擬機）。請確保 Podman socket 已啟動，以便 `podman-compose` 可以連線：

```bash
systemctl --user enable --now podman.socket
```
:::

</TabItem>
</Tabs>

### 系統資源

| 資源 | 最低需求 | 建議配置 |
|----------|---------|-------------|
| 磁碟空間 | 5 GB | 10 GB 以上 |
| 記憶體 | 分配 8 GB 給容器執行環境 | 12 GB 以上 |
| CPU | 2 核心 | 4 核心以上 |

<Tabs syncKey="runtime">
<TabItem label="Docker">

在 Docker Desktop &rarr; Settings &rarr; Resources 中調整資源限制。

</TabItem>
<TabItem label="Podman">

在 **macOS/Windows** 上，透過 Podman Desktop &rarr; Settings &rarr; Resources 調整資源限制，或透過 CLI：

```bash
podman machine set --cpus 4 --memory 12288
podman machine stop && podman machine start
```

在 **Linux** 上，容器直接共享主機資源——不需要額外的資源設定。

</TabItem>
</Tabs>

## 1. 設定

建立資料夾並下載 compose 檔案：

```bash
mkdir devcontainer && cd devcontainer
curl -fsSLO https://raw.githubusercontent.com/f5xc-salesdemos/devcontainer/main/docker-compose.yml
```

## 2. 新增您的驗證憑證（選用）

在同一資料夾中建立 `.env` 檔案以預先設定 Claude Code。請選擇以下其中一個選項：

### 選項 A：Claude Max（OAuth）

```ini
CLAUDE_CODE_OAUTH_TOKEN=sk-ant-oat01-your-token-here
```

### 選項 B：LiteLLM 或 Anthropic 相容代理

如果您有 LiteLLM 實例或其他 Anthropic 相容代理，請設定以下兩個變數：

```ini
LITELLM_BASE_URL=https://your-litellm-endpoint.example.com
LITELLM_API_KEY=your-api-key
```

只需設定網域——容器會自動衍生特定供應商的 URL 後綴（例如 Claude Code 使用 `/anthropic`）。

### 從本機憑證自動填入

如果您的主機上已設定 `git` 和 `gh`，您可以自動填入大部分 `.env` 的值。首先複製範本：

```bash
cp .env.example .env
```

然後執行與您設定相符的命令：

```bash
# Git identity
echo "GIT_AUTHOR_EMAIL=$(git config user.email)" >> .env
echo "GIT_AUTHOR_NAME=\"$(git config user.name)\"" >> .env

# Timezone
echo "TZ=$(readlink /etc/localtime | sed 's|.*/zoneinfo/||')" >> .env

# GitHub CLI token (requires: gh auth login)
echo "GH_TOKEN=$(gh auth token)" >> .env

# SSH key (if you use SSH for git)
echo "SSH_PRIVATE_KEY=$(base64 < ~/.ssh/id_ed25519)" >> .env
```

接著編輯 `.env` 以新增您的 OAuth 權杖或 LiteLLM 代理設定（請參閱上方選項）。

:::tip[AI 助手]
`.env.example` 檔案在每個可從本機工具填入的變數上都包含 `@auto-detect` 標記。像 Claude Code 這樣的 AI 助手可以掃描這些標記並執行相關命令，自動建立完整的 `.env`。
:::

## 3. 啟動

<Tabs syncKey="runtime">
<TabItem label="Docker">

```bash
docker compose up -d
```

</TabItem>
<TabItem label="Podman">

```bash
podman-compose pull && podman-compose up -d
```

</TabItem>
</Tabs>

首次執行會從 ghcr.io 拉取預建映像檔（依網路連線速度約需 1 分鐘）。後續啟動使用快取的映像檔，只需數秒。

## 6. 您的第一個專案

```bash
cd /workspace
git clone https://github.com/your-org/your-project.git
cd your-project
claude
```

## 7. 遠端顯示（noVNC）

容器內建虛擬顯示，可用於觀看 AI 代理控制瀏覽器。開啟 `http://localhost:6080/vnc.html` 即可連線。詳情請參閱[遠端顯示（noVNC）](./vnc)。

## 8. 瀏覽器自動化（Chrome DevTools MCP）

Claude Code 可以使用內建的 Chrome DevTools MCP 伺服器瀏覽網頁、擷取螢幕截圖並檢查 DOM。無需額外設定——它會在無頭模式下自動運作。詳情請參閱 [Chrome DevTools MCP](./chrome)。

## 9. 停止與重新啟動

<Tabs syncKey="runtime">
<TabItem label="Docker">

```bash
# 停止（保留資料）
docker compose down

# 再次啟動
docker compose up -d

# 銷毀所有資料並重新開始
docker compose down -v
docker compose up -d
```

</TabItem>
<TabItem label="Podman">

```bash
# 停止（保留資料）
podman-compose down

# 再次啟動（pull 確保取得最新映像檔）
podman-compose pull && podman-compose up -d

# 銷毀所有資料並重新開始
podman-compose down -v
podman-compose pull && podman-compose up -d
```

</TabItem>
</Tabs>

您在 `/workspace` 和家目錄中的程式碼在重新啟動後仍會保留。詳情請參閱[設定 — 資料持久化](./configuration#data-persistence)。
