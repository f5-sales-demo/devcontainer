:::tip[Installation rapide sur macOS]
Si vous êtes sur macOS avec Apple Silicon, ignorez cette page et utilisez l'[installateur en une ligne](/devcontainer/install/) :

```bash
curl -fsSL https://f5-sales-demo.github.io/devcontainer/scripts/install.sh | bash
```

Il gère les paquets Homebrew, la configuration d'iTerm2, la mise en place de Podman et télécharge tout ce dont vous avez besoin.
::::

## Prérequis

### Environnement d'exécution de conteneurs

Vous avez besoin d'un environnement d'exécution de conteneurs sur votre machine. Docker ou Podman fonctionnent tous les deux.

<Tabs syncKey="runtime">
<TabItem label="Docker">

| Plateforme | Installation |
|----------|---------|
| macOS | [Docker Desktop pour Mac](https://docs.docker.com/desktop/install/mac-install/) |
| Windows | [Docker Desktop pour Windows](https://docs.docker.com/desktop/install/windows-install/) (backend WSL2) |
| Linux | [Docker Engine](https://docs.docker.com/engine/install/) ou [Docker Desktop pour Linux](https://docs.docker.com/desktop/install/linux/) |

Vérifiez que tout fonctionne :

```bash
docker --version
docker compose version
```

</TabItem>
<TabItem label="Podman">

| Plateforme | Installation |
|----------|---------|
| macOS | [Podman Desktop pour Mac](https://podman-desktop.io/docs/installation/macos-install) |
| Windows | [Podman Desktop pour Windows](https://podman-desktop.io/docs/installation/windows-install) |
| Linux | [Podman](https://podman.io/docs/installation#installing-on-linux) + [Podman Compose](https://github.com/containers/podman-compose#installation) |

Vérifiez que tout fonctionne :

```bash
podman --version
podman-compose version
```

:::note[Linux uniquement]
Podman sur Linux exécute les conteneurs directement (sans VM). Assurez-vous que le socket Podman est actif pour que `podman-compose` puisse se connecter :

```bash
systemctl --user enable --now podman.socket
```
:::

<TabItem label="Podman">

Sur **macOS/Windows**, ajustez les limites de ressources dans Podman Desktop &rarr; Paramètres &rarr; Ressources, ou via la ligne de commande :

```bash
podman machine set --cpus 4 --memory 12288
podman machine stop && podman machine start
```

## 1. Mise en place

Créez un dossier et téléchargez le fichier compose :

```bash
mkdir devcontainer && cd devcontainer
curl -fsSLO https://raw.githubusercontent.com/f5-sales-demo/devcontainer/main/docker-compose.yml
```

## 2. Ajoutez vos identifiants d'authentification (Optionnel)

Créez un fichier `.env` dans le même dossier pour préconfigurer Claude Code. Choisissez l'une des options suivantes :

### Option A : Claude Max (OAuth)

```ini
CLAUDE_CODE_OAUTH_TOKEN=sk-ant-oat01-your-token-here
```

### Option B : LiteLLM ou proxy compatible Anthropic

Si vous disposez d'une instance LiteLLM ou d'un autre proxy compatible Anthropic, définissez ces deux variables :

```ini
LITELLM_BASE_URL=https://your-litellm-endpoint.example.com
LITELLM_API_KEY=your-api-key
```

Définissez uniquement le domaine — le conteneur dérive automatiquement les suffixes d'URL spécifiques au fournisseur (par exemple `/anthropic` pour Claude Code).

### Remplissage automatique depuis les identifiants locaux

Si vous avez `git` et `gh` configurés sur votre machine hôte, vous pouvez remplir automatiquement la plupart des valeurs du `.env`. Commencez par copier le modèle :

```bash
cp .env.example .env
```

Puis exécutez les commandes correspondant à votre configuration :

```bash
# Git identity
echo "GIT_AUTHOR_EMAIL=$(git config user.email)" >> .env
echo "GIT_AUTHOR_NAME=\"$(git config user.name)\"" >> .env

# Timezone
echo "TZ=$(readlink /etc/localtime | sed 's|.*/zoneinfo/||')" >> .env

# GitHub CLI token (requires: gh auth login)
echo "GH_TOKEN=$(gh auth token)" >> .env

# SSH key (if you use SSH for git)
echo "SSH_PRIVATE_KEY=$(base64 < ~/.ssh/id_ed25519)" >> .env
```

Ensuite, éditez le `.env` pour ajouter votre jeton OAuth ou vos paramètres de proxy LiteLLM (voir les options ci-dessus).

:::tip[Assistants IA]
Le fichier `.env.example` contient des marqueurs `@auto-detect` sur chaque variable pouvant être remplie à partir d'outils locaux. Les assistants IA comme Claude Code peuvent scanner ces marqueurs et exécuter les commandes associées pour construire automatiquement un `.env` complet.
:::

## 3. Démarrage

<Tabs syncKey="runtime">
<TabItem label="Docker">

```bash
docker compose up -d
```

</TabItem>
<TabItem label="Podman">

```bash
podman-compose pull && podman-compose up -d
```

</TabItem>
</Tabs>

Le premier lancement télécharge l'image pré-construite depuis ghcr.io (~1 min selon votre connexion). Les démarrages suivants utilisent l'image en cache et ne prennent que quelques secondes.

## 7. Affichage distant (noVNC)

Le conteneur inclut un affichage virtuel pour observer les agents IA contrôler un navigateur. Ouvrez `http://localhost:6080/vnc.html` pour vous connecter. Consultez [Affichage distant (noVNC)](./vnc) pour plus de détails.

## 8. Automatisation du navigateur (Chrome DevTools MCP)

Claude Code peut naviguer sur des pages web, prendre des captures d'écran et inspecter le DOM grâce au serveur Chrome DevTools MCP intégré. Aucune configuration nécessaire — il fonctionne automatiquement en mode headless. Consultez [Chrome DevTools MCP](./chrome) pour plus de détails.

## 9. Arrêt et redémarrage

<Tabs syncKey="runtime">
<TabItem label="Docker">

```bash
# Arrêt (préserve les données)
docker compose down

# Redémarrage
docker compose up -d

# Tout détruire et repartir de zéro
docker compose down -v
docker compose up -d
```

</TabItem>
<TabItem label="Podman">

```bash
# Arrêt (préserve les données)
podman-compose down

# Redémarrage (pull garantit la dernière image)
podman-compose pull && podman-compose up -d

# Tout détruire et repartir de zéro
podman-compose down -v
podman-compose pull && podman-compose up -d
```

</TabItem>
</Tabs>

Votre code dans `/workspace` et le répertoire personnel persistent entre les redémarrages. Consultez [Configuration — Persistance des données](./configuration#data-persistence) pour plus de détails.
