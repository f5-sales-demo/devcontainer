# Configuration

All configuration lives in the `.env` file at the project root. This file is gitignored — it never gets committed.

## Environment Variables

### Required

| Variable | Description | Example |
|----------|-------------|---------|
| `OPENAI_API_KEY` | API key for your AI provider | `sk-abc123...` |
| `OPENAI_BASE_URL` | Base URL of your API endpoint | `https://api.openai.com/v1` |

### User Identity

These ensure files created inside the container have correct ownership:

| Variable | Description | How to Set |
|----------|-------------|------------|
| `USER_UID` | Your host user ID | `id -u` |
| `USER_GID` | Your host group ID | `id -g` |
| `USERNAME` | Your host username | `whoami` |

### Git

| Variable | Description | How to Set |
|----------|-------------|------------|
| `GIT_AUTHOR_NAME` | Name for git commits | `git config user.name` |
| `GIT_AUTHOR_EMAIL` | Email for git commits | `git config user.email` |

### SSH

| Variable | Description | How to Set |
|----------|-------------|------------|
| `SSH_PRIVATE_KEY` | Base64-encoded private key | `base64 < ~/.ssh/id_ed25519` |

### Model Mapping

The proxy maps generic model names to your backend's model identifiers:

| Variable | Default | Used For |
|----------|---------|----------|
| `BIG_MODEL` | `gpt-4o` | Primary model for complex tasks |
| `MIDDLE_MODEL` | `gpt-4o` | General-purpose model |
| `SMALL_MODEL` | `gpt-4o-mini` | Fast model for simple tasks |

### Proxy Tuning

| Variable | Default | Description |
|----------|---------|-------------|
| `LOG_LEVEL` | `WARNING` | Proxy log verbosity (`DEBUG`, `INFO`, `WARNING`, `ERROR`) |
| `MAX_TOKENS_LIMIT` | `64000` | Maximum tokens per request |
| `REQUEST_TIMEOUT` | `120` | Request timeout in seconds |
| `ANTHROPIC_API_KEY` | `changeme` | Key passed from dev container to proxy |

## How Tools Are Installed

Tools are installed in three layers:

### Layer 1: Dockerfile (Image Build)

Runs once when you build the image. Installs system packages (`apt-get`), standalone binaries (actionlint, yt-dlp), and sets up the container user with UID/GID mapping.

### Layer 2: Devcontainer Features (Container Creation)

Applied by VS Code or the `devcontainer` CLI. These are [OCI-distributed packages](https://containers.dev/features) that install runtimes and tools:

| Source | Features |
|--------|----------|
| **Official** (`devcontainers/features`) | common-utils, git, github-cli, node, python, go, rust, java, terraform, kubectl-helm, docker-outside-of-docker, aws-cli, azure-cli, powershell |
| **Community** | claude-code, opencode, codex, uv, prettier, act, terraform-docs, devcontainers-cli |

### Layer 3: Post-Create Script

Runs after features are installed, using the runtimes they provide:

- **pip**: pre-commit, ansible, black, pylint, yamllint
- **npm**: markdownlint-cli2, openclaw

## Adding Tools

### Devcontainer Features

Edit `.devcontainer/devcontainer.json` and add entries to the `features` object:

```json
{
  "features": {
    "ghcr.io/devcontainers/features/ruby": {},
    "ghcr.io/dhoeric/features/google-cloud-cli": {}
  }
}
```

Browse available features at [containers.dev/features](https://containers.dev/features). Rebuild after adding: `docker compose up -d --build` (or in VS Code: **Dev Containers → Rebuild Container**).

### System Packages

Edit the `Dockerfile`:

```dockerfile
RUN apt-get update && apt-get install -y --no-install-recommends \
    your-package-here \
    && apt-get clean \
    && rm -rf /var/lib/apt/lists/*
```

### Python or npm Tools

Edit `.devcontainer/scripts/post-create.sh`:

```bash
pip install --user your-python-tool
npm install -g your-npm-tool
```

## VS Code Extensions

Extensions are configured in `.devcontainer/devcontainer.json` under `customizations.vscode.extensions` and installed automatically when opening in VS Code.

## Data Persistence

All data lives in Docker named volumes — no host directories are mounted:

| Volume | Mount Point | Contents |
|--------|-------------|----------|
| `workspace` | `/workspace` | Your cloned repositories and project files |
| `home` | `/home/<username>` | Shell history, tool configs, caches, SSH keys |

Both persist across container restarts and rebuilds.

| Action | Data |
|--------|------|
| `docker compose down` | Preserved |
| `docker compose up -d --build` | Preserved |
| `docker compose down -v` | **Deleted** |

## Port Mapping

| Host | Container | Service |
|------|-----------|---------|
| `127.0.0.1:18082` | `8082` | API proxy (localhost only) |

The proxy is bound to `127.0.0.1` so it's only accessible from your machine, not the network.
