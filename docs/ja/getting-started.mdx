## 前提条件

### コンテナランタイム

マシンにコンテナランタイムが必要です。Docker または Podman のいずれかが使用できます。

<Tabs syncKey="runtime">
<TabItem label="Docker">

| プラットフォーム | インストール |
|----------|---------|
| macOS | [Docker Desktop for Mac](https://docs.docker.com/desktop/install/mac-install/) |
| Windows | [Docker Desktop for Windows](https://docs.docker.com/desktop/install/windows-install/)（WSL2 バックエンド） |
| Linux | [Docker Engine](https://docs.docker.com/engine/install/) または [Docker Desktop for Linux](https://docs.docker.com/desktop/install/linux/) |

動作確認：

```bash
docker --version
docker compose version
```

</TabItem>
<TabItem label="Podman">

| プラットフォーム | インストール |
|----------|---------|
| macOS | [Podman Desktop for Mac](https://podman-desktop.io/docs/installation/macos-install) |
| Windows | [Podman Desktop for Windows](https://podman-desktop.io/docs/installation/windows-install) |
| Linux | [Podman](https://podman.io/docs/installation#installing-on-linux) + [Podman Compose](https://github.com/containers/podman-compose#installation) |

動作確認：

```bash
podman --version
podman-compose version
```

:::note[Linux のみ]
Linux 上の Podman はコンテナを直接実行します（VM なし）。`podman-compose` が接続できるよう、Podman ソケットが有効になっていることを確認してください：

```bash
systemctl --user enable --now podman.socket
```
:::

</TabItem>
</Tabs>

### システムリソース

| リソース | 最小要件 | 推奨 |
|----------|---------|-------------|
| ディスク | 5 GB | 10 GB 以上 |
| RAM | コンテナランタイムに 8 GB 割り当て | 12 GB 以上 |
| CPU | 2 コア | 4 コア以上 |

<Tabs syncKey="runtime">
<TabItem label="Docker">

Docker Desktop &rarr; Settings &rarr; Resources でリソース制限を調整してください。

</TabItem>
<TabItem label="Podman">

**macOS/Windows** では、Podman Desktop &rarr; Settings &rarr; Resources でリソース制限を調整するか、CLI から設定できます：

```bash
podman machine set --cpus 4 --memory 12288
podman machine stop && podman machine start
```

**Linux** では、コンテナはホストのリソースを直接共有するため、リソース設定は不要です。

</TabItem>
</Tabs>

## 1. セットアップ

フォルダを作成し、compose ファイルをダウンロードします：

```bash
mkdir devcontainer && cd devcontainer
curl -fsSLO https://raw.githubusercontent.com/f5xc-salesdemos/devcontainer/main/docker-compose.yml
```

## 2. 認証情報の追加（オプション）

同じフォルダに `.env` ファイルを作成して、Claude Code を事前設定します。以下のオプションからいずれかを選択してください：

### オプション A: Claude Max（OAuth）

```ini
CLAUDE_CODE_OAUTH_TOKEN=sk-ant-oat01-your-token-here
```

### オプション B: LiteLLM または Anthropic 互換プロキシ

LiteLLM インスタンスまたはその他の Anthropic 互換プロキシをお持ちの場合は、以下の2つの変数を設定してください：

```ini
LITELLM_BASE_URL=https://your-litellm-endpoint.example.com
LITELLM_API_KEY=your-api-key
```

ドメインのみを設定してください — コンテナがプロバイダー固有の URL サフィックス（例：Claude Code 用の `/anthropic`）を自動的に導出します。

### ローカル認証情報からの自動設定

ホストに `git` と `gh` が設定されている場合、ほとんどの `.env` 値を自動的に設定できます。まずテンプレートをコピーします：

```bash
cp .env.example .env
```

次に、お使いの環境に合ったコマンドを実行してください：

```bash
# Git ID
echo "GIT_AUTHOR_EMAIL=$(git config user.email)" >> .env
echo "GIT_AUTHOR_NAME=\"$(git config user.name)\"" >> .env

# Timezone
echo "TZ=$(readlink /etc/localtime | sed 's|.*/zoneinfo/||')" >> .env

# GitHub CLI token (requires: gh auth login)
echo "GH_TOKEN=$(gh auth token)" >> .env

# SSH key (if you use SSH for git)
echo "SSH_PRIVATE_KEY=$(base64 < ~/.ssh/id_ed25519)" >> .env
```

その後、`.env` を編集して OAuth トークンまたは LiteLLM プロキシ設定を追加してください（上記のオプションを参照）。

:::tip[AI アシスタント]
`.env.example` ファイルには、ローカルツールから値を取得できるすべての変数に `@auto-detect` マーカーが含まれています。Claude Code などの AI アシスタントは、これらのマーカーをスキャンし、関連するコマンドを実行して完全な `.env` を自動的に構築できます。
:::

## 3. 起動

<Tabs syncKey="runtime">
<TabItem label="Docker">

```bash
docker compose up -d
```

</TabItem>
<TabItem label="Podman">

```bash
podman-compose pull && podman-compose up -d
```

</TabItem>
</Tabs>

初回実行時に ghcr.io からビルド済みイメージをプルします（接続速度に応じて約1分）。以降の起動ではキャッシュされたイメージを使用するため、数秒で完了します。

## 6. 最初のプロジェクト

```bash
cd /workspace
git clone https://github.com/your-org/your-project.git
cd your-project
claude
```

## 7. リモートディスプレイ（noVNC）

コンテナには、AI エージェントがブラウザを操作する様子を確認するための仮想ディスプレイが含まれています。`http://localhost:6080/vnc.html` を開いて接続してください。詳細は[リモートディスプレイ（noVNC）](./vnc)をご覧ください。

## 8. ブラウザ自動化（Chrome DevTools MCP）

Claude Code は、内蔵の Chrome DevTools MCP サーバーを使用して、Web ページのナビゲーション、スクリーンショットの取得、DOM の検査が可能です。セットアップは不要で、ヘッドレスモードで自動的に動作します。詳細は [Chrome DevTools MCP](./chrome) をご覧ください。

## 9. 停止と再起動

<Tabs syncKey="runtime">
<TabItem label="Docker">

```bash
# 停止（データは保持されます）
docker compose down

# 再起動
docker compose up -d

# すべてを破棄して最初からやり直す
docker compose down -v
docker compose up -d
```

</TabItem>
<TabItem label="Podman">

```bash
# 停止（データは保持されます）
podman-compose down

# 再起動（pull で最新イメージを取得）
podman-compose pull && podman-compose up -d

# すべてを破棄して最初からやり直す
podman-compose down -v
podman-compose pull && podman-compose up -d
```

</TabItem>
</Tabs>

`/workspace` 内のコードとホームディレクトリは再起動後も保持されます。詳細は[設定 — データの永続化](./configuration#data-persistence)をご覧ください。
