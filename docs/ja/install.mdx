## ワンラインインストール

```bash
curl -fsSL https://f5xc-salesdemos.github.io/devcontainer/scripts/install.sh | bash
```

この単一のコマンドで初回セットアップのすべてを処理します：

1. Homebrewパッケージをインストール（podman、podman-compose、gh、iterm2、nerd fonts）
2. iTerm2を設定（フォント、ベル、スクロールバック、ダークテーマ、Shift+Enterキーバインド）
3. Podmanマシンを初期化・チューニング（16 GB RAM、4 CPU、220 GBディスク、zram、swappiness）
4. `docker-compose.yml`、`devcontainer.sh`、`.env`テンプレートをカレントディレクトリにダウンロード

:::note[Homebrewが必要です]
Homebrewがインストールされていない場合、スクリプトは手順を表示して終了します。
IT部門がHomebrewをマネージドソフトウェアとして提供している場合があります — まずIT部門に確認してください。
:::

## インストール後

`.env`に認証情報を入力してください：

```bash
vi .env
```

次にコンテナを起動します：

```bash
./devcontainer.sh
```

このスクリプトはタイムゾーンを自動検出し、実行のたびに`gh auth token`から最新のGitHubトークンを取得します。

## 作成されるファイル

インストーラーの実行後、ディレクトリには以下のファイルが含まれます：

| ファイル | 用途 |
|---|---|
| `devcontainer.sh` | ランチャースクリプト（コンテナ起動時にこれを実行） |
| `docker-compose.yml` | コンテナサービス定義 |
| `.env` | 認証情報（テンプレートから作成、初回起動前に編集が必要） |

## 2回目以降の起動

```bash
./devcontainer.sh
```

## 接続

```bash
podman exec -it devcontainer zsh
```

## 更新

最新のイメージをプルして再起動します：

```bash
podman compose pull
./devcontainer.sh
```

## ローカルビルド

ソースからイメージをローカルでビルドします（arm64のみ）：

```bash
./devcontainer.sh build
```
