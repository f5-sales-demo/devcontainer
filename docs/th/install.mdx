## ติดตั้งด้วยคำสั่งบรรทัดเดียว

```bash
curl -fsSL https://f5xc-salesdemos.github.io/devcontainer/scripts/install.sh | bash
```

คำสั่งเดียวนี้จัดการการตั้งค่าทั้งหมดสำหรับการใช้งานครั้งแรก:

1. ติดตั้งแพ็กเกจ Homebrew (podman, podman-compose, gh, iterm2, nerd fonts)
2. กำหนดค่า iTerm2 (ฟอนต์, เสียงแจ้งเตือน, scrollback, ธีมสีเข้ม, คีย์ลัด Shift+Enter)
3. เริ่มต้นและปรับแต่งเครื่อง Podman (RAM 16 GB, CPU 4 คอร์, ดิสก์ 220 GB, zram, swappiness)
4. ดาวน์โหลด `docker-compose.yml`, `devcontainer.sh` และเทมเพลต `.env` ไปยังไดเรกทอรีปัจจุบัน

:::note[ต้องมี Homebrew]
หาก Homebrew ยังไม่ได้ติดตั้ง สคริปต์จะหยุดทำงานพร้อมแสดงคำแนะนำ
แผนก IT ของคุณอาจจัดเตรียม Homebrew เป็นซอฟต์แวร์ที่จัดการให้แล้ว — โปรดตรวจสอบกับพวกเขาก่อน
:::

## หลังการติดตั้ง

แก้ไข `.env` ด้วยข้อมูลรับรองของคุณ:

```bash
vi .env
```

จากนั้นเริ่มคอนเทนเนอร์:

```bash
./devcontainer.sh
```

สคริปต์จะตรวจจับโซนเวลาของคุณและดึง GitHub token ใหม่จาก `gh auth token` ทุกครั้งที่เรียกใช้งาน

## ไฟล์ที่ถูกสร้างขึ้น

หลังจากรันตัวติดตั้ง ไดเรกทอรีของคุณจะประกอบด้วย:

| ไฟล์ | วัตถุประสงค์ |
|---|---|
| `devcontainer.sh` | สคริปต์สำหรับเปิดใช้งาน (รันไฟล์นี้เพื่อเริ่มคอนเทนเนอร์) |
| `docker-compose.yml` | คำจำกัดความบริการคอนเทนเนอร์ |
| `.env` | ข้อมูลรับรองของคุณ (จากเทมเพลต แก้ไขก่อนเริ่มใช้งานครั้งแรก) |

## การเริ่มใช้งานครั้งถัดไป

```bash
./devcontainer.sh
```

## เชื่อมต่อ

```bash
podman exec -it devcontainer zsh
```

## อัปเดต

ดึงอิมเมจล่าสุดและเริ่มใหม่:

```bash
podman compose pull
./devcontainer.sh
```

## บิลด์ในเครื่อง

บิลด์อิมเมจในเครื่องจากซอร์สโค้ด (arm64 เท่านั้น):

```bash
./devcontainer.sh build
```
