## ติดตั้งด้วยคำสั่งเดียว

```bash
curl -fsSL https://f5xc-salesdemos.github.io/devcontainer/scripts/install.sh | bash
```

คำสั่งเดียวนี้จัดการการตั้งค่าครั้งแรกทั้งหมด:

1. ติดตั้งแพ็กเกจ Homebrew (podman, podman-compose, gh, iterm2, nerd fonts)
2. กำหนดค่า iTerm2 (ฟอนต์, เสียงเตือน, scrollback, ธีมมืด, การกำหนดคีย์ Shift+Enter)
3. เริ่มต้นและปรับแต่ง Podman machine (RAM 16 GB, 4 CPUs, ดิสก์ 220 GB, zram, swappiness)
4. ดาวน์โหลด `docker-compose.yml`, `devcontainer.sh` และเทมเพลต `.env` ไปยังไดเรกทอรีปัจจุบัน

:::note[ต้องมี Homebrew]
หาก Homebrew ยังไม่ได้ติดตั้ง สคริปต์จะหยุดทำงานพร้อมแสดงคำแนะนำ
แผนก IT ของคุณอาจจัดเตรียม Homebrew เป็นซอฟต์แวร์ที่มีการจัดการ — กรุณาตรวจสอบกับพวกเขาก่อน
:::

## หลังการติดตั้ง

แก้ไข `.env` ด้วยข้อมูลรับรองของคุณ:

```bash
vi .env
```

จากนั้นเริ่มต้นคอนเทนเนอร์:

```bash
./devcontainer.sh
```

สคริปต์จะตรวจจับเขตเวลาของคุณและดึง GitHub token ใหม่จาก `gh auth token` ในทุกครั้งที่เรียกใช้

## สิ่งที่ถูกสร้างขึ้น

หลังจากรันตัวติดตั้ง ไดเรกทอรีของคุณจะประกอบด้วย:

| ไฟล์ | วัตถุประสงค์ |
|---|---|
| `devcontainer.sh` | สคริปต์ตัวเปิด (รันไฟล์นี้เพื่อเริ่มคอนเทนเนอร์) |
| `docker-compose.yml` | คำจำกัดความของบริการคอนเทนเนอร์ |
| `.env` | ข้อมูลรับรองของคุณ (จากเทมเพลต แก้ไขก่อนเริ่มใช้งานครั้งแรก) |

## การเริ่มต้นครั้งถัดไป

```bash
./devcontainer.sh
```

## เชื่อมต่อ

```bash
podman exec -it devcontainer zsh
```

## อัปเดต

ดึงอิมเมจล่าสุดและรีสตาร์ท:

```bash
podman compose pull
./devcontainer.sh
```

## บิลด์ในเครื่อง

บิลด์อิมเมจในเครื่องจากซอร์สโค้ด (เฉพาะ arm64):

```bash
./devcontainer.sh build
```
