คอนเทนเนอร์จะรันสแต็กจอแสดงผลเสมือน (Xvfb + x11vnc + noVNC + fluxbox) เพื่อให้คุณสามารถดู AI agents ควบคุมเบราว์เซอร์และเข้าแทรกแซงเมื่อจำเป็น — เช่น เพื่อจัดการหน้าล็อกอินหรือพรอมต์ MFA

## การเชื่อมต่อ

1. เปิด `http://localhost:6080/vnc.html` ในเบราว์เซอร์ของคุณแล้วคลิก **Connect**
2. คุณจะเห็นเดสก์ท็อป fluxbox ที่เบา — นี่คือจอแสดงผลเสมือนของคอนเทนเนอร์

AI agents ที่ใช้ `@playwright/mcp` ในโหมดมีหน้าจอจะเรนเดอร์ Chromium ไปยังจอแสดงผลนี้ คุณสามารถคลิกและพิมพ์โดยตรงในหน้าต่าง noVNC เพื่อเข้าแทรกแซงได้ตลอดเวลา — ไม่จำเป็นต้องหยุดชั่วคราว/กลับมาทำงานต่อ

## การเปิดเบราว์เซอร์ด้วยตนเอง

เพื่อเปิดเบราว์เซอร์ด้วยตนเองภายในคอนเทนเนอร์:

```bash
chromium --no-sandbox &
```

เพื่อรันการทดสอบ Playwright ในโหมดมีหน้าจอ:

```bash
npx playwright test --headed
```

## ตัวแปรสภาพแวดล้อม

| ตัวแปร | ค่าเริ่มต้น | คำอธิบาย |
|----------|---------|-------------|
| `ENABLE_VNC` | `true` | ตั้งเป็น `false` เพื่อปิดสแต็ก VNC |
| `VNC_RESOLUTION` | `1280x1024x24` | ความละเอียดจอแสดงผลเสมือน (กว้างxสูงxความลึก) |
| `DISPLAY` | `:99` | หมายเลขจอแสดงผล X11 |
| `NOVNC_HOST_PORT` | `6080` | พอร์ตฝั่งโฮสต์สำหรับ noVNC (เปลี่ยนเพื่อหลีกเลี่ยงการขัดแย้งกับคอนเทนเนอร์อื่น) |

ตั้งค่าเหล่านี้ในไฟล์ `.env` หรือใน `.devcontainer/devcontainer.json` ภายใต้ `containerEnv`

## การปิดสแต็ก VNC

สำหรับการทำงานแบบ headless เท่านั้น ให้ตั้ง `ENABLE_VNC=false` ใน `.env` แล้วรีสตาร์ทคอนเทนเนอร์:

<Tabs syncKey="runtime">
<TabItem label="Docker">

```bash
docker compose down
docker compose up -d
```

</TabItem>
<TabItem label="Podman">

```bash
podman-compose down
podman-compose up -d
```

</TabItem>
</Tabs>
