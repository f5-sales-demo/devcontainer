หากคุณใช้ macOS กับ Apple Silicon ให้ข้ามหน้านี้แล้วใช้ [ตัวติดตั้งบรรทัดเดียว](/devcontainer/install/):

ตัวติดตั้งจะจัดการแพ็กเกจ Homebrew, การตั้งค่า iTerm2, การตั้งค่า Podman และดาวน์โหลดทุกอย่างที่คุณต้องการ

## ข้อกำหนดเบื้องต้น

### Container Runtime

คุณต้องมี container runtime บนเครื่องของคุณ Docker หรือ Podman สามารถใช้ได้ทั้งสอง

<Tabs syncKey="runtime">
<TabItem label="Docker">

| แพลตฟอร์ม | การติดตั้ง |
|----------|---------|
| macOS | [Docker Desktop for Mac](https://docs.docker.com/desktop/install/mac-install/) |
| Windows | [Docker Desktop for Windows](https://docs.docker.com/desktop/install/windows-install/) (WSL2 backend) |
| Linux | [Docker Engine](https://docs.docker.com/engine/install/) หรือ [Docker Desktop for Linux](https://docs.docker.com/desktop/install/linux/) |

ตรวจสอบว่าทำงานได้:

```bash
docker --version
docker compose version
```

</TabItem>
<TabItem label="Podman">

| แพลตฟอร์ม | การติดตั้ง |
|----------|---------|
| macOS | [Podman Desktop for Mac](https://podman-desktop.io/docs/installation/macos-install) |
| Windows | [Podman Desktop for Windows](https://podman-desktop.io/docs/installation/windows-install) |
| Linux | [Podman](https://podman.io/docs/installation#installing-on-linux) + [Podman Compose](https://github.com/containers/podman-compose#installation) |

ตรวจสอบว่าทำงานได้:

```bash
podman --version
podman-compose version
```

:::note[เฉพาะ Linux]
Podman บน Linux รันคอนเทนเนอร์โดยตรง (ไม่มี VM) ตรวจสอบให้แน่ใจว่า Podman socket ทำงานอยู่เพื่อให้ `podman-compose` สามารถเชื่อมต่อได้:

```bash
systemctl --user enable --now podman.socket
```
:::

</TabItem>
</Tabs>

## 2. เพิ่มข้อมูลรับรองการยืนยันตัวตน (ไม่บังคับ)

สร้างไฟล์ `.env` ในโฟลเดอร์เดียวกันเพื่อตั้งค่า Claude Code ล่วงหน้า เลือกหนึ่งในตัวเลือกต่อไปนี้:

### ตัวเลือก A: Claude Max (OAuth)

```ini
CLAUDE_CODE_OAUTH_TOKEN=sk-ant-oat01-your-token-here
```

### ตัวเลือก B: LiteLLM หรือ proxy ที่เข้ากันได้กับ Anthropic

หากคุณมี LiteLLM instance หรือ proxy อื่นที่เข้ากันได้กับ Anthropic ให้ตั้งค่าตัวแปรสองตัวนี้:

```ini
LITELLM_BASE_URL=https://your-litellm-endpoint.example.com
LITELLM_API_KEY=your-api-key
```

ตั้งค่าเฉพาะโดเมน — คอนเทนเนอร์จะสร้าง URL suffix เฉพาะผู้ให้บริการโดยอัตโนมัติ (เช่น `/anthropic` สำหรับ Claude Code)

### ดึงข้อมูลอัตโนมัติจากข้อมูลรับรองในเครื่อง

หากคุณมี `git` และ `gh` ที่ตั้งค่าไว้บนเครื่องโฮสต์ คุณสามารถกรอกค่า `.env` ส่วนใหญ่ได้โดยอัตโนมัติ เริ่มต้นด้วยการคัดลอกเทมเพลต:

```bash
cp .env.example .env
```

จากนั้นรันคำสั่งที่ตรงกับการตั้งค่าของคุณ:

```bash
# Git identity
echo "GIT_AUTHOR_EMAIL=$(git config user.email)" >> .env
echo "GIT_AUTHOR_NAME=\"$(git config user.name)\"" >> .env

# Timezone
echo "TZ=$(readlink /etc/localtime | sed 's|.*/zoneinfo/||')" >> .env

# GitHub CLI token (requires: gh auth login)
echo "GH_TOKEN=$(gh auth token)" >> .env

# SSH key (if you use SSH for git)
echo "SSH_PRIVATE_KEY=$(base64 < ~/.ssh/id_ed25519)" >> .env
```

จากนั้นแก้ไข `.env` เพื่อเพิ่ม OAuth token หรือการตั้งค่า LiteLLM proxy (ดูตัวเลือกด้านบน)

:::tip[ผู้ช่วย AI]
ไฟล์ `.env.example` มีเครื่องหมาย `@auto-detect` ในทุกตัวแปรที่สามารถกรอกจากเครื่องมือในเครื่องได้ ผู้ช่วย AI เช่น Claude Code สามารถสแกนเครื่องหมายเหล่านี้และรันคำสั่งที่เกี่ยวข้องเพื่อสร้างไฟล์ `.env` ที่สมบูรณ์โดยอัตโนมัติ
:::

## 7. การแสดงผลระยะไกล (noVNC)

คอนเทนเนอร์มี virtual display สำหรับดู AI agent ควบคุมเบราว์เซอร์ เปิด `http://localhost:6080/vnc.html` เพื่อเชื่อมต่อ ดู [การแสดงผลระยะไกล (noVNC)](./vnc) สำหรับรายละเอียด

## 8. การควบคุมเบราว์เซอร์อัตโนมัติ (Chrome DevTools MCP)

Claude Code สามารถนำทางหน้าเว็บ ถ่ายภาพหน้าจอ และตรวจสอบ DOM โดยใช้เซิร์ฟเวอร์ Chrome DevTools MCP ในตัว ไม่ต้องตั้งค่า — ทำงานโดยอัตโนมัติในโหมด headless ดู [Chrome DevTools MCP](./chrome) สำหรับรายละเอียด

## 9. การหยุดและเริ่มต้นใหม่

<Tabs syncKey="runtime">
<TabItem label="Docker">

```bash
# หยุด (เก็บรักษาข้อมูล)
docker compose down

# เริ่มต้นอีกครั้ง
docker compose up -d

# ลบทุกอย่างและเริ่มต้นใหม่
docker compose down -v
docker compose up -d
```

</TabItem>
<TabItem label="Podman">

```bash
# หยุด (เก็บรักษาข้อมูล)
podman-compose down

# เริ่มต้นอีกครั้ง (pull เพื่อให้แน่ใจว่าได้อิมเมจล่าสุด)
podman-compose pull && podman-compose up -d

# ลบทุกอย่างและเริ่มต้นใหม่
podman-compose down -v
podman-compose pull && podman-compose up -d
```

</TabItem>
</Tabs>

โค้ดของคุณใน `/workspace` และไดเรกทอรี home จะคงอยู่แม้รีสตาร์ท ดู [การตั้งค่า — การคงอยู่ของข้อมูล](./configuration#data-persistence) สำหรับรายละเอียด
