## ข้อกำหนดเบื้องต้น

### Container Runtime

คุณต้องมี container runtime บนเครื่องของคุณ Docker หรือ Podman สามารถใช้ได้ทั้งคู่

<Tabs syncKey="runtime">
<TabItem label="Docker">

| แพลตฟอร์ม | ติดตั้ง |
|----------|---------|
| macOS | [Docker Desktop for Mac](https://docs.docker.com/desktop/install/mac-install/) |
| Windows | [Docker Desktop for Windows](https://docs.docker.com/desktop/install/windows-install/) (WSL2 backend) |
| Linux | [Docker Engine](https://docs.docker.com/engine/install/) หรือ [Docker Desktop for Linux](https://docs.docker.com/desktop/install/linux/) |

ตรวจสอบว่าทำงานได้:

```bash
docker --version
docker compose version
```

</TabItem>
<TabItem label="Podman">

| แพลตฟอร์ม | ติดตั้ง |
|----------|---------|
| macOS | [Podman Desktop for Mac](https://podman-desktop.io/docs/installation/macos-install) |
| Windows | [Podman Desktop for Windows](https://podman-desktop.io/docs/installation/windows-install) |
| Linux | [Podman](https://podman.io/docs/installation#installing-on-linux) + [Podman Compose](https://github.com/containers/podman-compose#installation) |

ตรวจสอบว่าทำงานได้:

```bash
podman --version
podman-compose version
```

:::note[เฉพาะ Linux]
Podman บน Linux รันคอนเทนเนอร์โดยตรง (ไม่ต้องใช้ VM) ตรวจสอบให้แน่ใจว่า Podman socket ทำงานอยู่เพื่อให้ `podman-compose` สามารถเชื่อมต่อได้:

```bash
systemctl --user enable --now podman.socket
```
:::

</TabItem>
</Tabs>

## 2. เพิ่มข้อมูลรับรองการยืนยันตัวตน (ไม่บังคับ)

สร้างไฟล์ `.env` ในโฟลเดอร์เดียวกันเพื่อตั้งค่า Claude Code ล่วงหน้า เลือกหนึ่งในตัวเลือกต่อไปนี้:

### ตัวเลือก A: Claude Max (OAuth)

```ini
CLAUDE_CODE_OAUTH_TOKEN=sk-ant-oat01-your-token-here
```

### ตัวเลือก B: LiteLLM หรือ Anthropic-compatible proxy

หากคุณมี LiteLLM instance หรือ Anthropic-compatible proxy อื่น ให้ตั้งค่าตัวแปรสองตัวนี้:

```ini
LITELLM_BASE_URL=https://your-litellm-endpoint.example.com
LITELLM_API_KEY=your-api-key
```

ตั้งค่าเฉพาะโดเมน — คอนเทนเนอร์จะสร้าง URL suffix เฉพาะผู้ให้บริการโดยอัตโนมัติ (เช่น `/anthropic` สำหรับ Claude Code)

### ดึงค่าอัตโนมัติจากข้อมูลรับรองในเครื่อง

หากคุณมี `git` และ `gh` ตั้งค่าไว้บนเครื่องโฮสต์แล้ว คุณสามารถกรอกค่า `.env` ส่วนใหญ่ได้โดยอัตโนมัติ เริ่มจากคัดลอกเทมเพลต:

```bash
cp .env.example .env
```

จากนั้นรันคำสั่งที่ตรงกับการตั้งค่าของคุณ:

```bash
# Git identity
echo "GIT_AUTHOR_EMAIL=$(git config user.email)" >> .env
echo "GIT_AUTHOR_NAME=\"$(git config user.name)\"" >> .env

# Timezone
echo "TZ=$(readlink /etc/localtime | sed 's|.*/zoneinfo/||')" >> .env

# GitHub CLI token (requires: gh auth login)
echo "GH_TOKEN=$(gh auth token)" >> .env

# SSH key (if you use SSH for git)
echo "SSH_PRIVATE_KEY=$(base64 < ~/.ssh/id_ed25519)" >> .env
```

จากนั้นแก้ไข `.env` เพื่อเพิ่ม OAuth token หรือการตั้งค่า LiteLLM proxy (ดูตัวเลือกด้านบน)

:::tip[ผู้ช่วย AI]
ไฟล์ `.env.example` มีเครื่องหมาย `@auto-detect` บนทุกตัวแปรที่สามารถดึงค่าจากเครื่องมือในเครื่องได้ ผู้ช่วย AI เช่น Claude Code สามารถสแกนเครื่องหมายเหล่านี้และรันคำสั่งที่เกี่ยวข้องเพื่อสร้าง `.env` ที่สมบูรณ์โดยอัตโนมัติ
:::

## 7. จอแสดงผลระยะไกล (noVNC)

คอนเทนเนอร์มีจอแสดงผลเสมือนสำหรับดู AI agent ควบคุมเบราว์เซอร์ เปิด `http://localhost:6080/vnc.html` เพื่อเชื่อมต่อ ดูรายละเอียดที่ [จอแสดงผลระยะไกล (noVNC)](./vnc)

## 8. ระบบอัตโนมัติสำหรับเบราว์เซอร์ (Chrome DevTools MCP)

Claude Code สามารถเปิดหน้าเว็บ ถ่ายภาพหน้าจอ และตรวจสอบ DOM โดยใช้ Chrome DevTools MCP server ในตัว ไม่ต้องตั้งค่าใดๆ — ทำงานโดยอัตโนมัติในโหมด headless ดูรายละเอียดที่ [Chrome DevTools MCP](./chrome)

## 9. หยุดและเริ่มใหม่

<Tabs syncKey="runtime">
<TabItem label="Docker">

```bash
# หยุด (เก็บรักษาข้อมูล)
docker compose down

# เริ่มอีกครั้ง
docker compose up -d

# ลบทุกอย่างและเริ่มใหม่ตั้งแต่ต้น
docker compose down -v
docker compose up -d
```

</TabItem>
<TabItem label="Podman">

```bash
# หยุด (เก็บรักษาข้อมูล)
podman-compose down

# เริ่มอีกครั้ง (pull เพื่อรับ image ล่าสุด)
podman-compose pull && podman-compose up -d

# ลบทุกอย่างและเริ่มใหม่ตั้งแต่ต้น
podman-compose down -v
podman-compose pull && podman-compose up -d
```

</TabItem>
</Tabs>

โค้ดของคุณใน `/workspace` และ home directory จะคงอยู่ข้ามการรีสตาร์ท ดูรายละเอียดที่ [การตั้งค่า — การคงอยู่ของข้อมูล](./configuration#data-persistence)
