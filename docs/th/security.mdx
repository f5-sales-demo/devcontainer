## ความเป็นมา

ในเดือนกุมภาพันธ์ 2025 LayerX Security Research ได้เปิดเผยช่องโหว่ร้ายแรงแบบ zero-click remote code execution ในเฟรมเวิร์ก Claude Desktop Extensions (DXT) ของ Anthropic ส่วนขยาย (Extensions) ทำงานโดยไม่มีแซนด์บ็อกซ์ด้วยสิทธิ์ระบบเต็มรูปแบบ ทำหน้าที่เป็นสะพานเชื่อมการทำงานระหว่างโมเดลภาษาและระบบปฏิบัติการในเครื่อง

ทีมรักษาความปลอดภัยขององค์กรหลายแห่งกำหนดให้เครื่องมือเขียนโค้ด AI ต้องทำงานภายในเครื่องเสมือน (virtual machines) คอนเทนเนอร์ หรือสภาพแวดล้อมแบบใช้แล้วทิ้งเท่านั้น — ไม่ใช่บนอุปกรณ์ปลายทางขององค์กรโดยตรง

**devcontainer นี้ตอบสนองข้อกำหนดดังกล่าว**

## โมเดลการแยกส่วน

```
+------------------------------------------------+
|  Host Machine (corporate endpoint)             |
|                                                |
|  - No AI tool binaries installed               |
|  - No AI tool extensions or plugins            |
|  - Only Docker is required                     |
|                                                |
|  +------------------------------------------+  |
|  |  Docker Container (isolated)             |  |
|  |                                          |  |

|  |  x Other host applications               |  |
|  +------------------------------------------+  |
+------------------------------------------------+
```

## การควบคุมความปลอดภัย

### ไม่มีการเมาท์โวลุ่มจากโฮสต์

ข้อมูลทั้งหมดอยู่ใน Docker named volumes — ไม่ใช่ไดเรกทอรีของโฮสต์แบบ bind-mount การถูกบุกรุกภายในคอนเทนเนอร์**ไม่สามารถ**เข้าถึงไฟล์บนเครื่องโฮสต์ของคุณได้ ดู [การกำหนดค่า — การเก็บรักษาข้อมูล](./configuration#data-persistence) สำหรับรายละเอียดเกี่ยวกับโวลุ่ม

### ไม่มีที่เก็บข้อมูลรับรองร่วมกัน

คอนเทนเนอร์ไม่มีสิทธิ์เข้าถึง macOS Keychain, Windows Credential Manager, โปรไฟล์เบราว์เซอร์ หรือ SSH agent ของโฮสต์ SSH keys จะถูกฉีดเข้ามาตอนเริ่มต้นผ่านตัวแปรสภาพแวดล้อมที่เข้ารหัสด้วย base64 และมีอยู่เฉพาะภายในคอนเทนเนอร์เท่านั้น

### ไม่มีการซิงค์คลิปบอร์ด

คอนเทนเนอร์ Docker ไม่แชร์สถานะคลิปบอร์ดกับโฮสต์ ข้อมูลไม่สามารถรั่วไหลผ่านการคัดลอก/วางได้

### การแยกส่วนเครือข่าย

คอนเทนเนอร์สื่อสารกับโลกภายนอกในโหมดการยืนยันตัวตนที่รองรับสองโหมด:

- **โหมด LiteLLM proxy** — HTTPS ขาออกไปยัง LiteLLM proxy ของคุณผ่าน NAT ของ Docker
- **โหมด OAuth** — HTTPS ขาออกตรงไปยัง Anthropic ผ่าน NAT ของ Docker

คอนเทนเนอร์ไม่มีสิทธิ์เข้าถึง VPN tunnels, อินเตอร์เฟซเครือข่ายขององค์กร หรือบริการอื่นๆ บนโฮสต์

### ออกแบบมาให้ใช้ชั่วคราว

สภาพแวดล้อมทั้งหมดสามารถถูกทำลายและสร้างใหม่ได้ในไม่กี่วินาที ปฏิบัติต่อคอนเทนเนอร์เสมือนเป็นสิ่งที่ใช้แล้วทิ้ง

### เครื่องมือแบบ CLI เท่านั้น

สภาพแวดล้อมนี้เรียกใช้เครื่องมือ AI เป็น**อินเตอร์เฟซบรรทัดคำสั่งเท่านั้น** ไม่มีแอปพลิเคชันเดสก์ท็อป ไม่มีส่วนขยายเบราว์เซอร์ ไม่มีเฟรมเวิร์ก DXT ช่องโหว่ที่เป็นแรงจูงใจในการสร้างโมเดลการแยกส่วนนี้ไม่มีผลกับการใช้งาน CLI ภายในคอนเทนเนอร์

### การเสริมความแข็งแกร่งของคอนเทนเนอร์

`docker-compose.yml` ใช้มาตรการป้องกันเชิงลึกหลายประการ:

- **การยกเลิก Capability** — Linux capabilities ทั้งหมดถูกยกเลิก (`cap_drop: ALL`) จากนั้นเพิ่มกลับเฉพาะชุดขั้นต่ำ: `CHOWN`, `DAC_OVERRIDE`, `FOWNER` (สำหรับแก้ไขสิทธิ์โวลุ่มใน entrypoint), `SETUID`/`SETGID` (สำหรับ sudo), `NET_RAW` (สำหรับ ping/tcpdump) และ `NET_ADMIN` (สำหรับเครื่องมือดักจับแพ็กเก็ตเช่น Wireshark และ bettercap)
- **การจำกัดทรัพยากร** — CPU (4 คอร์), หน่วยความจำ (8 GB) และจำนวน PID (4096) ถูกจำกัดผ่าน `mem_limit`, `cpus` และ `pids_limit` เพื่อป้องกันการโจมตีแบบทรัพยากรหมด
- **tmpfs สำหรับ `/tmp`** — ไดเรกทอรี `/tmp` ถูกเมาท์เป็น tmpfs ขนาด 256 MB เก็บไฟล์ชั่วคราวในหน่วยความจำและป้องกันไม่ให้คงอยู่ในเลเยอร์ที่เขียนได้ของคอนเทนเนอร์
- **การตรวจสอบ TLS เปิดใช้งานโดยค่าเริ่มต้น** — `NODE_TLS_REJECT_UNAUTHORIZED` ไม่ได้ถูกตั้งค่า ดังนั้น Node.js จะตรวจสอบใบรับรอง TLS โดยค่าเริ่มต้น ผู้ใช้ที่ต้องการยอมรับใบรับรองที่ลงนามเอง (เช่น สำหรับ internal proxy) สามารถเลือกเปิดใช้งานได้โดยเพิ่ม `NODE_TLS_REJECT_UNAUTHORIZED=0` ในไฟล์ `.env`
- **SSH key umask** — SSH private keys ถูกเขียนภายใต้ `umask 077` ดังนั้นไฟล์จะไม่สามารถอ่านได้โดยทุกคน — แม้แต่ช่วงเวลาสั้นๆ — ระหว่างการสร้างและการเรียก `chmod`

### เครื่องมือทดสอบความปลอดภัยและการเจาะระบบ

คอนเทนเนอร์มีเครื่องมือทดสอบความปลอดภัยและการเจาะระบบที่ติดตั้งไว้ล่วงหน้าประมาณ 80 รายการสำหรับสถานการณ์การทดสอบที่ได้รับอนุญาต เครื่องมือถูกจัดเรียงตามหมวดหมู่:

| หมวดหมู่ | เครื่องมือ (ตัวอย่าง) |
|----------|-----------------|
| วิเคราะห์เครือข่าย | tshark, wireshark, masscan, hping3, bettercap (amd64), netdiscover |
| สแกนเนอร์เว็บ | nikto, sqlmap, dirb, whatweb, sslscan, OWASP ZAP, dalfox, feroxbuster |
| รหัสผ่านและการยืนยันตัวตน | hydra, john, hashcat, medusa, ncrack |
| วิศวกรรมย้อนกลับ | radare2, Ghidra, gdb, binwalk, strace, ltrace |
| การสำรวจ | subfinder, amass, httpx, nuclei, gau, waybackurls, recon-ng, spiderfoot |
| การ Fuzz และการระบุรายการ | ffuf, gobuster, SecLists |
| ซัพพลายเชนและความลับ | trufflehog, grype, syft, gitleaks, trivy |
| เฟรมเวิร์กการโจมตี | Metasploit (amd64), searchsploit (ExploitDB) |

เครื่องมือบางตัว (bettercap, Metasploit) มีให้ใช้งานเฉพาะบน amd64 เนื่องจากข้อจำกัดของแพ็กเกจต้นทาง เครื่องมือดักจับแพ็กเก็ต (tshark, bettercap) ต้องการ capability `NET_ADMIN` ซึ่งรวมอยู่ใน `docker-compose.yml` ค่าเริ่มต้น

สำหรับรายการทั้งหมด ให้รัน `claude-self-test` ภายในคอนเทนเนอร์ (ส่วนที่ 7 ตรวจสอบเครื่องมือความปลอดภัยทั้งหมด)

## รายการตรวจสอบการปฏิบัติตามข้อกำหนด

| ข้อกำหนด | สถานะ |
|---|---|
| ไม่ได้ติดตั้งเครื่องมือ AI บนอุปกรณ์ปลายทางของโฮสต์ | ใช่ |
| ทำงานในเครื่องเสมือนหรือคอนเทนเนอร์ที่แยกส่วน | ใช่ |
| ไม่มีการเข้าถึง file shares ของโฮสต์ | ใช่ |
| ไม่มีการซิงค์คลิปบอร์ด | ใช่ |
| ไม่มีที่เก็บข้อมูลรับรองร่วมกัน | ใช่ |
| ไม่มี VPN split tunneling เข้าเครือข่าย production | ใช่ |
| สภาพแวดล้อมใช้แล้วทิ้งได้ | ใช่ |
| ไม่มีส่วนขยายเดสก์ท็อปหรือปลั๊กอิน | ใช่ |
| Linux capabilities ถูกยกเลิก (สิทธิ์ขั้นต่ำ) | ใช่ |
| การจำกัดทรัพยากรถูกบังคับใช้ (CPU/หน่วยความจำ/PIDs) | ใช่ |
| การตรวจสอบใบรับรอง TLS เปิดใช้งานโดยค่าเริ่มต้น | ใช่ |
| บริการภายในผูกกับ localhost เท่านั้น | ใช่ |

## แนวทางปฏิบัติที่ดีที่สุด

1. **อย่าติดตั้งเครื่องมือเขียนโค้ด AI บนเครื่องโฮสต์ของคุณ** ใช้คอนเทนเนอร์นี้แทน
2. **อย่าเพิ่ม bind mounts** ใน `docker-compose.yml` การกำหนดค่า named-volume เป็นไปตามเจตนา
3. **หมุนเวียน SSH keys** ที่ใช้ภายในคอนเทนเนอร์เป็นระยะ
4. **ตรวจสอบ `.env` ก่อนแชร์** ไฟล์นี้มี API keys และอาจมี SSH private keys
5. **อัปเดตเป็นระยะ** เพื่อรับแพตช์ความปลอดภัย:
