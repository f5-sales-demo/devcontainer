:::tip[macOS 快速安装]
如果您使用的是搭载 Apple Silicon 的 macOS，请跳过本页面并使用[一行命令安装器](/devcontainer/install/)：

```bash
curl -fsSL https://f5-sales-demo.github.io/devcontainer/scripts/install.sh | bash
```

它会处理 Homebrew 包、iTerm2 配置、Podman 设置，并下载您所需的一切。
::::

## 前置要求

### 容器运行时

您需要在机器上安装容器运行时。Docker 或 Podman 均可。

<Tabs syncKey="runtime">
<TabItem label="Docker">

| 平台 | 安装方式 |
|----------|---------|
| macOS | [Docker Desktop for Mac](https://docs.docker.com/desktop/install/mac-install/) |
| Windows | [Docker Desktop for Windows](https://docs.docker.com/desktop/install/windows-install/)（WSL2 后端） |
| Linux | [Docker Engine](https://docs.docker.com/engine/install/) 或 [Docker Desktop for Linux](https://docs.docker.com/desktop/install/linux/) |

验证是否正常工作：

```bash
docker --version
docker compose version
```

</TabItem>
<TabItem label="Podman">

| 平台 | 安装方式 |
|----------|---------|
| macOS | [Podman Desktop for Mac](https://podman-desktop.io/docs/installation/macos-install) |
| Windows | [Podman Desktop for Windows](https://podman-desktop.io/docs/installation/windows-install) |
| Linux | [Podman](https://podman.io/docs/installation#installing-on-linux) + [Podman Compose](https://github.com/containers/podman-compose#installation) |

验证是否正常工作：

```bash
podman --version
podman-compose version
```

:::note[仅限 Linux]
Linux 上的 Podman 直接运行容器（无需虚拟机）。请确保 Podman socket 处于活动状态，以便 `podman-compose` 可以连接：

```bash
systemctl --user enable --now podman.socket
```
:::

</TabItem>
</Tabs>

### 系统资源

| 资源 | 最低要求 | 推荐配置 |
|----------|---------|-------------|
| 磁盘 | 5 GB | 10 GB+ |
| 内存 | 分配给容器运行时 8 GB | 12 GB+ |
| CPU | 2 核 | 4+ 核 |

<Tabs syncKey="runtime">
<TabItem label="Docker">

在 Docker Desktop &rarr; Settings &rarr; Resources 中调整资源限制。

</TabItem>
<TabItem label="Podman">

在 **macOS/Windows** 上，通过 Podman Desktop &rarr; Settings &rarr; Resources 或命令行调整资源限制：

```bash
podman machine set --cpus 4 --memory 12288
podman machine stop && podman machine start
```

在 **Linux** 上，容器直接共享宿主机资源 — 无需资源配置。

</TabItem>
</Tabs>

## 1. 设置

创建文件夹并下载 compose 文件：

```bash
mkdir devcontainer && cd devcontainer
curl -fsSLO https://raw.githubusercontent.com/f5-sales-demo/devcontainer/main/docker-compose.yml
```

## 2. 添加认证凭据（可选）

在同一文件夹中创建 `.env` 文件以预配置 Claude Code。请选择以下选项之一：

### 选项 A：Claude Max（OAuth）

```ini
CLAUDE_CODE_OAUTH_TOKEN=sk-ant-oat01-your-token-here
```

### 选项 B：LiteLLM 或 Anthropic 兼容代理

如果您有 LiteLLM 实例或其他 Anthropic 兼容代理，请设置以下两个变量：

```ini
LITELLM_BASE_URL=https://your-litellm-endpoint.example.com
LITELLM_API_KEY=your-api-key
```

只需设置域名 — 容器会自动派生特定提供商的 URL 后缀（例如 Claude Code 的 `/anthropic`）。

### 从本地凭据自动填充

如果您的宿主机上已配置 `git` 和 `gh`，可以自动填充大部分 `.env` 值。首先复制模板：

```bash
cp .env.example .env
```

然后运行与您设置匹配的命令：

```bash
# Git 身份信息
echo "GIT_AUTHOR_EMAIL=$(git config user.email)" >> .env
echo "GIT_AUTHOR_NAME=\"$(git config user.name)\"" >> .env

# 时区
echo "TZ=$(readlink /etc/localtime | sed 's|.*/zoneinfo/||')" >> .env

# GitHub CLI 令牌（需要先执行：gh auth login）
echo "GH_TOKEN=$(gh auth token)" >> .env

# SSH 密钥（如果您使用 SSH 进行 git 操作）
echo "SSH_PRIVATE_KEY=$(base64 < ~/.ssh/id_ed25519)" >> .env
```

然后编辑 `.env` 添加您的 OAuth 令牌或 LiteLLM 代理设置（参见上述选项）。

:::tip[AI 助手]
`.env.example` 文件在每个可以从本地工具填充的变量上都包含 `@auto-detect` 标记。像 Claude Code 这样的 AI 助手可以扫描这些标记并运行相关命令，自动构建完整的 `.env` 文件。
:::

## 3. 启动

<Tabs syncKey="runtime">
<TabItem label="Docker">

```bash
docker compose up -d
```

</TabItem>
<TabItem label="Podman">

```bash
podman-compose pull && podman-compose up -d
```

</TabItem>
</Tabs>

首次运行会从 ghcr.io 拉取预构建镜像（根据网络情况约需 1 分钟）。后续启动使用缓存镜像，仅需几秒。

```bash
cd /workspace

claude
```

## 7. 远程显示（noVNC）

容器包含虚拟显示器，用于观看 AI 代理控制浏览器。打开 `http://localhost:6080/vnc.html` 进行连接。详情参见[远程显示（noVNC）](./vnc)。

## 8. 浏览器自动化（Chrome DevTools MCP）

Claude Code 可以使用内置的 Chrome DevTools MCP 服务器浏览网页、截取屏幕截图和检查 DOM。无需设置 — 它在无头模式下自动工作。详情参见 [Chrome DevTools MCP](./chrome)。

## 9. 停止和重启

<Tabs syncKey="runtime">
<TabItem label="Docker">

```bash
# 停止（保留数据）
docker compose down

# 再次启动
docker compose up -d

# 销毁所有内容并重新开始
docker compose down -v
docker compose up -d
```

</TabItem>
<TabItem label="Podman">

```bash
# 停止（保留数据）
podman-compose down

# 再次启动（pull 确保使用最新镜像）
podman-compose pull && podman-compose up -d

# 销毁所有内容并重新开始
podman-compose down -v
podman-compose pull && podman-compose up -d
```

</TabItem>
</Tabs>

您在 `/workspace` 和主目录中的代码在重启后会持久保留。详情参见[配置 — 数据持久化](./configuration#data-persistence)。
